## Proyectos Destacados 📂

Aquí presento mis proyectos clave, la mayoría desarrollados en entornos colaborativos usando SCRUM, Conventional Commits y buenas prácticas de Git. Cada uno incluye documentación detallada en su README, con instrucciones de instalación, uso y contribución. Aplico principios SOLID y arquitecturas sólidas para asegurar escalabilidad.

### Edusync
- **Descripción**: Plataforma para consumir APIs en tiempo real, visualizando el valor de Bitcoin con gráficos, conversiones y un CRUD inteligente para guardar datos.
- **Tecnologías**: JavaScript puro, Chart.js, MongoDB, Node.js, Express.js
- **Repositorio**: [github.com/sebasBetancourt/Edusync](https://github.com/sebasBetancourt/Edusync)
- **Destacado**: Uso de APIs externas para datos en vivo y operaciones CRUD.

### OrbisProCLI
- **Descripción**: Ejecutador de comandos en Node.js para freelancers, organizando proyectos y clientes con CRUDS personalizados en bases de datos.
- **Tecnologías**: JavaScript puro, MongoDB, Node.js, dotenv
- **Repositorio**: [github.com/sebasBetancourt/Orbis-ProCLI](https://github.com/sebasBetancourt/Orbis-ProCLI.git)
- **Destacado**: Interfaz CLI eficiente para gestión de datos.

### CampusParking
- **Descripción**: Sistema de gestión de parqueaderos para registrar ingresos/salidas de vehículos en múltiples sedes, zonas y tipos.
- **Tecnologías**: JavaScript puro, MongoDB, SCRUM
- **Repositorio**: [github.com/sebasBetancourt/Campus-Parking](https://github.com/sebasBetancourt/Campus-Parking)
- **Destacado**: Aplicación de patrones de diseño y principios SOLID; trabajo colaborativo con evidencias de SCRUM.

### GameCampus
- **Descripción**: Plataforma web interactiva inspirada en Dungeons & Dragons, donde usuarios crean personajes personalizables con razas, clases, habilidades y equipos.
- **Tecnologías**: JavaScript puro, Consumo de APIs, SCRUM, HTML, CSS
- **Repositorio**: [github.com/julianort11/GameCampus](https://github.com/julianort11/GameCampus)
- **Destacado**: Proyecto colaborativo con integración de APIs y elementos de juego.

| Proyecto | Tecnologías Principales | Enfoque | Colaborativo |
|----------|--------------------------|---------|--------------|
| Edusync | JS, Chart.js, MongoDB, Node.js | APIs en tiempo real, Gráficos | Sí |
| OrbisProCLI | JS, MongoDB, Node.js | CLI y CRUDS | Sí |
| CampusParking | JS, MongoDB, SCRUM | Gestión de datos, SOLID | Sí |

Todos los proyectos siguen Conventional Commits con emojis para una historia clara de progresos (ej: `feat: ✨ Agregar funcionalidad CRUD`). Incluyen documentación de APIs (si aplican), despliegues en GitHub Pages o Vercel donde sea frontend, y evidencias de trabajo en equipo (ramas, pulls, issues en SCRUM).

## Documentación General 📚

- **Instalación y Uso**: Cada proyecto tiene un README detallado con pasos para clonar, instalar dependencias (`npm install` o similar) y ejecutar (`npm start`).
- **Colaboración**: ¡Bienvenido a contribuir! Usa ramas feature/ para nuevas funcionalidades. Sigue Conventional Commits: `feat:`, `fix:`, `docs:`, etc.
- **Arquitectura**: Aplico MVC donde aplica, principios SOLID para código mantenible y patrones como Singleton o Factory.
- **Licencia**: MIT – Libre para usar y modificar.

Para más detalles, revisa los README individuales. Uso versiones semánticas en Git para un flujo claro.
